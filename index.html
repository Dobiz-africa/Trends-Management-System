👤 Users

users
  • idUUIDPrimary keyPK
  • nameVARCHAR(100)Full name
  • emailVARCHAR(255)Unique email address
  • roleENUMadmin, finance, gis, md
  • teamVARCHAR(50)Team A, Team B, Mochudi (nullable)
  • is_activeBOOLEANDefault true
  • created_atTIMESTAMPDefault now()
🔗 work_orders (issued_by) · documents (created_by) · signatures (user_id)

📋 Work Orders

work_orders
  • idUUIDPrimary keyPK
  • wo_numberVARCHAR(50)BPC Work Order number (unique)UNIQUE
  • customer_nameVARCHAR(255)Customer full name
  • locationVARCHAR(255)Town / Village / Ward
  • connection_typeVARCHAR(50)U/G Conn, OHL 400V, etc.
  • phaseVARCHAR(10)Phase 46, 47, etc.
  • location_factorDECIMAL(5,2)Default 29.25%
  • markupDECIMAL(5,2)Default 12.5%
  • current_stageENUMwo_received, vo1_created, linesman_sent, field_received, vo2_created, teams_notified, work_complete, gis_notified, gis_complete, claim_docs_ready, claim_submitted
  • claim_refVARCHAR(50)Certificate reference (TES-001, etc.)
  • created_byUUIDAdmin who createdFK → users
  • created_atTIMESTAMPDefault now()

📄 Documents

documents
  • idUUIDPrimary keyPK
  • work_order_idUUIDAssociated work orderFK → work_orders
  • document_typeENUMbpc_wo, vo1, vo2, field_report, works_instruction, gis_report, annexure, payment_cert, invoice, list_of_jobs
  • file_urlTEXTSupabase storage path
  • dataJSONBStructured document data (line items, calculations)
  • statusENUMpending, scanned, signed, submitted
  • created_byUUIDCreatorFK → users
  • signed_byUUIDSigner (hand signature recorded)FK → users
  • signed_atTIMESTAMPWhen hand signature was scanned
  • created_atTIMESTAMPDefault now()
🔗 line_items (document_id) · signatures (document_id)

📝 Line Items

document_line_items
  • idUUIDPrimary keyPK
  • document_idUUIDParent documentFK → documents
  • rate_idUUIDReference to rate (optional)FK → rates
  • descriptionTEXTService description
  • unitVARCHAR(20)Ea, M, m3, etc.
  • quantityDECIMAL(10,2)Quantity
  • rateDECIMAL(12,2)Unit rate
  • valueDECIMAL(12,2)Auto-calculated (qty × rate)
  • sort_orderINTDisplay order

💰 Rates Sheet

rates
  • idUUIDPrimary keyPK
  • descriptionTEXTService description (autocomplete search)
  • unitVARCHAR(20)Ea, M, m3, etc.
  • rateDECIMAL(12,2)Unit rate in BWP
  • categoryVARCHAR(50)U/G Service Connection, Overhead Line, etc.
  • is_activeBOOLEANOnly active rates shown in autocomplete
  • effective_dateDATEWhen rate became effective

🔔 Notifications

notifications
  • idUUIDPrimary keyPK
  • user_idUUIDRecipientFK → users
  • work_order_idUUIDRelated work orderFK → work_orders
  • typeVARCHAR(50)gis_assignment, claim_ready, vo2_review, etc.
  • messageTEXTNotification message
  • is_readBOOLEANDefault false
  • created_atTIMESTAMPDefault now()

📎 Scans / Uploads

scans
  • idUUIDPrimary keyPK
  • document_idUUIDRelated documentFK → documents
  • file_urlTEXTSupabase storage path
  • file_nameVARCHAR(255)Original filename
  • uploaded_byUUIDWho uploadedFK → users
  • uploaded_atTIMESTAMPDefault now()
🔐 Role-Based Access Control (RLS Policies)

🛡️ Admin

Full access to:

  • Create/Read/Update/Delete work orders
  • Create VO1, VO2 documents
  • Record all external actions (linesman, teams, GIS notification)
  • Upload signed scans for all document types
  • Mark claim as submitted to BPC
  • Manage rates sheet (activate/deactivate rates)

💰 Finance

Limited access to:

  • View jobs where stage = gis_complete
  • Select jobs for claim batch
  • Generate Annexure, Payment Certificate, Invoice, List of Jobs Done
  • Upload signed scans for claim documents
  • No pipeline view No activity log

📍 GIS Consultant

Limited access to:

  • View only jobs where stage = gis_notified
  • Fill in GIS report form
  • Print GIS report → hand sign → scan → upload
  • Submit GIS report (advances stage to gis_complete)
  • Only GIS stages visible Only GIS report document

👑 Managing Director

Full oversight access:

  • See ALL work orders with progress percentage
  • Full workflow pipeline for each job (all 11 stages)
  • Download ALL documents (scans and generated)
  • Activity log visible (every action tracked)
  • Print claim package to take to BPC
  • Read-only No editing
🔄 Workflow Stages (11 Steps)
1. Work Order Received from BPC → 2. VO1 Created → 3. VO1 Sent to Linesman → 4. Field Findings Received → 5. VO2 Created → 6. Teams Notified → 7. Work Complete → 8. GIS Notified → 9. GIS Report Uploaded → 10. Claim Documents Generated → 11. Claim Submitted to BPC
📑 Document Types & Ownership

Generated by Admin

  • Works Valuation (VO1)
  • Variation Order (VO2)
  • Field Report (entered from linesman findings)
  • Works Instruction

Generated by GIS

  • GIS Geo-Analysis Report

Generated by Finance

  • Annexure to Payment Certificate
  • Payment Certificate
  • Invoice
  • List of Jobs Done

Uploaded by Anyone

  • BPC Work Order (original scan)
  • All hand-signed document scans
📌 Schema Summary
Total Tables: 7 (users, work_orders, documents, document_line_items, rates, notifications, scans)
Primary Keys: UUID for all tables (Supabase native)
Foreign Keys: work_orders.created_by → users.id, documents.work_order_id → work_orders.id, documents.created_by → users.id, document_line_items.document_id → documents.id, etc.
RLS Enabled: Row Level Security policies per role
Storage: Supabase Storage for PDF/image uploads (scans)
Real-time: Supabase Realtime for notifications
